<objective>
Finalize WSL bootstrap support with a diagnostics script and focused documentation so that running the CLI under WSL reliably surfaces how Cursor, `clip.exe`, and workspace paths are resolved, and guides users to configure `GSD_CURSOR_BIN` and `GSD_CLIP_EXE` correctly.

Purpose: Make it trivial to diagnose and configure WSL-specific path and clipboard behavior so the autonomous daemon can run correctly in WSL without manual trial-and-error.
Output: A WSL bootstrap shell script, WSL-focused documentation, and README entry points that help users confirm their WSL environment is correctly resolved.
</objective>

**Constraints and intent:**
- Keep behavior non-destructive and focused on path resolution and diagnostics.
- Assume users may run the CLI directly or via `npx` from WSL shells.
- Prefer environment-variable overrides (`GSD_CURSOR_BIN`, `GSD_CLIP_EXE`) when present.
- Treat clipboard integration as optional: when `clip.exe` cannot be resolved, diagnostics should surface this clearly without failing the CLI.
