    Install tree-kill: `npm install tree-kill && npm install -D @types/tree-kill`

    Create `src/agent-runner.ts` with:

    1. `RunAgentOptions` interface: `{ agentPath: string, workspace: string, prompt: string, env?: Record<string,string>, timeoutMs?: number, model?: string, resumeId?: string, onEvent?: (event: CursorStreamEvent) => void }`

    2. `RunAgentResult` interface: `{ sessionId: string | null, resultEvent: ResultEvent | null, events: CursorStreamEvent[], exitCode: number | null, stderr: string }`

    3. `runAgent(options: RunAgentOptions): Promise<RunAgentResult>` function:
       - Build args array: `['-p', '--force', '--trust', '--approve-mcps', '--workspace', workspace, '--output-format', 'stream-json']`
       - If model specified, add `['--model', model]`
       - If resumeId specified, add `['--resume', resumeId]`
       - Append prompt as final arg
       - Spawn with `child_process.spawn(agentPath, args, { stdio: ['pipe','pipe','pipe'], env: { ...process.env, ...env } })`
       - Wire readline on child.stdout, parse each line with parseEvent() from stream-events.ts
       - Collect events array, call onEvent callback for each parsed event
       - Collect stderr via child.stderr data events
       - On close: resolve with RunAgentResult (use extractSessionId/extractResult helpers)

    4. `abortAgent(child: ChildProcess): Promise<void>` — exported helper that calls tree-kill on child.pid with SIGTERM, falls back to SIGKILL after 5s

    Use spawn() arrays for args (never string concatenation — avoids shell injection). Do NOT use shell: true.
  </action>

  <done>agent-runner.ts exports runAgent and abortAgent functions, tree-kill installed, types compile cleanly</done>

    4. `getLastRunningSession(logPath: string): Promise<SessionLogEntry | null>` — find last entry with status 'running' (for Phase 5 crash recovery)
