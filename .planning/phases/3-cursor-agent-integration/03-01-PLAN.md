<feature>
  <name>NDJSON Stream Event Parser</name>
  <files>src/stream-events.ts, src/stream-events.test.ts</files>
  <behavior>
    parseEvent(line: string) → CursorStreamEvent | null

    Valid inputs (JSON string → typed event):
    - '{"type":"system","subtype":"init","session_id":"abc","model":"claude-4","cwd":"/tmp","apiKeySource":"env","permissionMode":"auto"}' → SystemInitEvent
    - '{"type":"assistant","message":{"role":"assistant","content":[{"type":"text","text":"hello"}]},"session_id":"abc"}' → AssistantEvent
    - '{"type":"tool_call","subtype":"started","call_id":"c1","tool_call":{"name":"Shell"},"session_id":"abc"}' → ToolCallEvent
    - '{"type":"tool_call","subtype":"completed","call_id":"c1","tool_call":{"name":"Shell"},"session_id":"abc"}' → ToolCallEvent
    - '{"type":"result","subtype":"success","duration_ms":5000,"duration_api_ms":4500,"is_error":false,"result":"done","session_id":"abc"}' → ResultEvent

    Edge cases:
    - Malformed JSON (not valid JSON) → returns null (no throw)
    - Valid JSON but unknown event type → returns null (ignore unknown types per spec)
    - Valid JSON with extra unknown fields → still parses successfully (Zod passthrough)
    - Empty string → returns null

    Helper: extractSessionId(events: CursorStreamEvent[]) → string | null
    - Returns session_id from first SystemInitEvent, or null if none found

    Helper: extractResult(events: CursorStreamEvent[]) → ResultEvent | null
    - Returns the result event, or null if not present
  </behavior>
  <implementation>
    Define Zod schemas for each event type using z.object() with .passthrough() to tolerate unknown fields.
    Use z.discriminatedUnion('type', [...]) for the top-level CursorStreamEvent type.
    parseEvent wraps JSON.parse + schema.safeParse — returns null on any failure.
    Export TypeScript types inferred from Zod schemas.

    Test framework setup (RED phase): Install vitest as devDependency. Create vitest.config.ts with ts paths.
    Add "test" script to package.json.
  </implementation>
</feature>
