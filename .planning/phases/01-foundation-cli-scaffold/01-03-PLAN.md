<task type="auto">
  <name>Task 2: Daemon loop skeleton and CLI wiring</name>
  <files>src/daemon.ts, src/cli.ts</files>
  <action>
Create src/daemon.ts with:

1. Export async function runDaemon(config: AutopilotConfig, logger: Logger):
   - Load goals from config.goalsPath using loadGoals()
   - Get pending goals using getPendingGoals()
   - If no pending goals, log "No pending goals in queue" and return
   - Log goal count: "Found {n} pending goals"
   - If config.parallel:
     - Log "Parallel mode: processing up to {config.maxConcurrent} goals concurrently"
     - (Stub: just log each goal title, don't actually parallelize yet)
   - If sequential:
     - Log "Sequential mode: processing goals one at a time"
   - For each pending goal:
     - Log "Processing goal: {title}"
     - Call stub processGoal(goal, config, logger) which just logs "TODO: orchestrate goal via cursor-agent" and returns
     - Log "Completed goal: {title}" (stub — real completion in Phase 2)
   - Log "All goals processed"

2. Register graceful shutdown:
   - Listen for SIGINT and SIGTERM
   - On signal, log "Shutting down gracefully..."
   - Set a flag to stop processing after current goal
   - If second signal received, force exit

3. Update src/cli.ts action handler:
   - Remove the stub log from Plan 01
   - Initialize logger based on --verbose flag
   - Call loadConfig() with configPath from --config and CLI overrides from parsed flags
   - If --dry-run: load goals, print summary table (title, status), exit
   - Otherwise: call runDaemon(config, logger)
   - Wrap in try/catch: log error and exit(1) on failure
  </action>
  <verify>npm run build succeeds, ./bin/gsd-autopilot --goals goals.md --verbose shows goal processing, --dry-run shows summary table, Ctrl+C triggers graceful shutdown log</verify>
