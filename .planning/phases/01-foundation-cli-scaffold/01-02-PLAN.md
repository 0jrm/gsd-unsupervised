<task type="auto">
  <name>Task 2: Goals.md parser</name>
  <files>src/goals.ts</files>
  <action>
Create src/goals.ts with:

1. Define Goal type:
   - title: string
   - status: "pending" | "in_progress" | "done"
   - raw: string (original markdown line for round-tripping)

2. Export function parseGoals(markdown: string): Goal[]:
   - Split by sections: find ## Pending, ## In Progress, ## Done headers
   - Within each section, match lines starting with "- [ ]" (pending/in_progress) or "- [x]" (done)
   - Extract title by stripping the checkbox prefix and trimming
   - Assign status based on which section the line appears in
   - Ignore blank lines, comments (<!-- -->), and lines that don't match checkbox pattern
   - Return array of all goals across sections

3. Export function loadGoals(filePath: string): Promise<Goal[]>:
   - Read file from disk (fs.readFile)
   - If file doesn't exist, throw with message: "Goals file not found: {path}"
   - Call parseGoals on contents
   - If no pending goals found, log warning (don't error — empty queue is valid)
   - Return parsed goals

4. Export function getPendingGoals(goals: Goal[]): Goal[]:
   - Filter to status === "pending"

Edge cases to handle:
- Empty file → return []
- File with headers but no goals → return []
- Goals with special characters in title → preserve as-is
- Windows line endings (\r\n) → normalize to \n before parsing
  </action>
  <verify>npm run build succeeds, parseGoals with the project's actual goals.md returns 2 pending goals with correct titles</verify>
  <done>Parser extracts goals from all three sections, handles edge cases, loadGoals reads from disk, getPendingGoals filters correctly</done>
</task>
