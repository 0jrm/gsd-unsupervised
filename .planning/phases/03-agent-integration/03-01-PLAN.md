Ensure `runAgent` in `src/agent-runner.ts` is the single spawning seam for the Cursor agent, accepting options for agent path, workspace, prompt, env, timeout, model, and resume ID, and always invoking the binary with:
- `-p --force --trust --approve-mcps` flags for non-interactive, trusted, auto-approved execution.
- `--workspace &lt;dir&gt;` pointing at the autopilot workspace root.
- `--output-format stream-json` so stdout is line-delimited JSON.

Use `node:child_process.spawn` with stdio `['pipe','pipe','pipe']`, wrap stdout in a `readline` interface, and feed each line through `parseEvent` from `src/stream-events.ts`, collecting a list of typed `CursorStreamEvent` values and forwarding them to an optional `onEvent` callback.

Capture stderr into an in-memory string buffer; when the child exits, return an object containing `sessionId` (extracted via `extractSessionId`), the last `result` event (via `extractResult`), the full events array, the exit code, a `timedOut` boolean, and the stderr text (plus timeout diagnostics if applicable).

Add a timeout path that, when `timeoutMs &gt; 0`, schedules a timer to mark the run as timed out and call `abortAgent` (implemented with `tree-kill`), ensuring the promise still resolves with `timedOut: true` once the process actually closes. Do not reject on timeout; callers should treat timeout as a structured error case.

Keep the implementation defensive: catch spawn errors and reject with a clear message that includes the agent path; avoid throwing from event handlers.

  <action>
Implement `createCursorAgentInvoker` and `createAgentInvoker` in `src/cursor-agent.ts` so that:
- `createCursorAgentInvoker` accepts a `CursorAgentConfig` (agent binary path, default timeout, session log path, optional heartbeat path/interval), wraps a call to `runAgent`, and adapts the result into the `AgentInvoker` contract (`success`, `output`/`error` message).
- The invoker writes a "running" `SessionLogEntry` to the configured JSONL log before starting, including goal title, phase and plan numbers, and the full GSD command string.
- While the agent runs, if `heartbeatPath` is configured, it writes an ISO timestamp to that file immediately and then on a fixed interval (`heartbeatIntervalMs`) using `fs.promises.writeFile`, and ensures the heartbeat file is unlinked and the interval cleared when the run finishes or throws.
- On success (exit code 0 with a result event), it appends a "done" entry with duration and session ID; on timeout, it appends a "timeout" entry with duration and an explanatory error; on non-timeout failures or thrown errors, it appends a "crashed" entry with duration and error details, always best-effort.

Implement `createAgentInvoker` as a small factory that chooses between the Cursor implementation and stubbed adapters for other `AgentId` values, using `getCursorBinaryPath(config)` and `config.agentTimeoutMs` to wire the cursor case, and returning fast-resolving stubs (logging-only) for unsupported agents until real adapters exist.
