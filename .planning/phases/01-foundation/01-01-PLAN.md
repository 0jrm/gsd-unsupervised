<task type="auto">
  <name>Task 1: Implement Commander-based CLI entrypoint</name>
  <files>src/cli.ts</files>
  <action>
    Use Commander to define the `gsd-unsupervised` CLI with options that match the project requirements:
    - `--goals &lt;path&gt;` (default `./goals.md`) to specify the goals markdown file.
    - `--config &lt;path&gt;` (default `./.autopilot/config.json`) for the daemon config JSON.
    - `--parallel` and `--max-concurrent &lt;n&gt;` to control multi-goal execution.
    - `--verbose` to enable debug logging and pretty pino output.
    - `--dry-run` to parse goals and print a queue summary without starting the daemon.
    - `--agent &lt;name&gt;`, `--agent-path &lt;path&gt;`, and `--agent-timeout &lt;ms&gt;` to select and configure the agent runner.
    - `--status-server &lt;port&gt;` to enable the HTTP status server.

    The action body should:
    - Read the package version from `package.json` for `program.version()`.
    - Initialize the root pino logger via `initLogger`, deriving level from `--verbose`.
    - Call `loadConfig` with the CLI overrides so config/goal paths and agent options are resolved consistently.
    - In `--dry-run` mode, load goals via `loadGoals(config.goalsPath)`, compute pending goals with `getPendingGoals`, and print a simple tabular summary (title + status, totals) to stdout, then exit without starting the daemon.
    - In normal mode, register shutdown handlers via `registerShutdownHandlers(logger)` and then call `runDaemon(config, logger)`; on any fatal error, log with `log.error` and exit non-zero.

    Pay attention to ESM patterns (`fileURLToPath`, `dirname`, `resolve`) so that version lookup and relative paths work when compiled to `dist/cli.js`. Avoid any interactive prompts; all behavior must be driven from CLI flags and config only.
  </action>
  <verify>
    - Run `npm run build`.
    - Execute `node dist/cli.js --help` and confirm that all documented options appear with correct defaults.
    - Run `node dist/cli.js --dry-run --goals ./goals.md` and confirm it prints a goals queue summary and exits with status code 0 without starting the daemon.
  </verify>
  <done>
    - `npm run build` succeeds with no TypeScript or runtime errors.
    - `node dist/cli.js --help` shows the expected options and description.
    - `node dist/cli.js --dry-run` against the default `goals.md` prints a table of goals and exits cleanly without running the daemon.
  </done>
</task>
