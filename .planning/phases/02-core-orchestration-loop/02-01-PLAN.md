**GSD command sequence for a goal:**
1. `/gsd/new-project` — initialize .planning/ directory
2. `/gsd/create-roadmap` — create ROADMAP.md with phases
3. For each phase N:
   a. `/gsd/plan-phase N` — create PLAN.md files in phase directory
   b. For each plan M in phase N:
      `/gsd/execute-plan .planning/phases/XX-name/NN-MM-PLAN.md` — execute plan, create SUMMARY.md
4. After all phases complete → goal done

**Key constraint:** Phase 3 will replace the agent stub with real cursor-agent invocation. The state machine must expose clean interfaces that Phase 3 can wire into without modifying lifecycle logic.

  <action>
Create src/lifecycle.ts with:

1. **GoalLifecyclePhase enum** (string enum for readability in logs):
   - `new` — goal parsed, not yet started
   - `initializing_project` — running /gsd/new-project
   - `creating_roadmap` — running /gsd/create-roadmap
   - `planning_phase` — running /gsd/plan-phase N
   - `executing_plan` — running /gsd/execute-plan path
   - `phase_complete` — all plans in current phase have summaries
   - `complete` — all phases done, goal finished
   - `failed` — unrecoverable error

2. **GsdCommand interface:**
   ```
   {
     command: string;        // e.g. "/gsd/new-project"
     args?: string;          // e.g. "2" for plan-phase, or full path for execute-plan
     description: string;    // human-readable for logging
   }
   ```

3. **GoalProgress interface:**
   ```
   {
     goalTitle: string;
     phase: GoalLifecyclePhase;
     currentPhaseNumber: number;    // 0 if not yet at phases
     totalPhases: number;           // 0 if roadmap not yet parsed
     currentPlanIndex: number;      // 0-based within current phase
     totalPlansInPhase: number;     // 0 if phase not yet planned
     lastCommand?: GsdCommand;
     error?: string;
   }
   ```

4. **LIFECYCLE_TRANSITIONS map** — a Record<GoalLifecyclePhase, GoalLifecyclePhase[]> defining which phases can follow which. This prevents invalid state jumps:
   - new → [initializing_project]
   - initializing_project → [creating_roadmap]
   - creating_roadmap → [planning_phase]
   - planning_phase → [executing_plan]
   - executing_plan → [executing_plan, phase_complete] (next plan or phase done)
   - phase_complete → [planning_phase, complete] (next phase or all done)
   - complete → [] (terminal)
   - failed → [] (terminal)

Export all types and the transitions map.

<task type="auto">
  <name>Task 2: State machine implementation</name>
  <files>src/lifecycle.ts</files>
  <action>
Add to src/lifecycle.ts:

1. **Export class GoalStateMachine:**

   Constructor takes `goalTitle: string`, initializes GoalProgress with phase=new, all counters at 0.

   **Methods:**

   a. `getProgress(): GoalProgress` — returns current progress snapshot (defensive copy)

   b. `getPhase(): GoalLifecyclePhase` — returns current phase

   c. `isComplete(): boolean` — returns true if phase is 'complete'

   d. `isFailed(): boolean` — returns true if phase is 'failed'

   e. `isTerminal(): boolean` — returns true if complete or failed

   f. `advance(to: GoalLifecyclePhase): void` — transition to new phase. Validate against LIFECYCLE_TRANSITIONS — if transition is not allowed, throw Error with message showing current phase, attempted phase, and allowed transitions. Update internal progress.

   g. `setPhaseInfo(currentPhase: number, totalPhases: number): void` — set phase tracking numbers (called after roadmap parsing)

   h. `setPlanInfo(currentPlan: number, totalPlans: number): void` — set plan tracking within current phase (called after plan discovery)

   i. `getNextCommand(): GsdCommand | null` — the core logic. Based on current phase and progress, return the next GSD command to execute:
      - `new` → return { command: '/gsd/new-project', description: 'Initialize project' }
      - `initializing_project` → return { command: '/gsd/create-roadmap', description: 'Create roadmap' }
      - `creating_roadmap` → return { command: '/gsd/plan-phase', args: String(currentPhaseNumber), description: `Plan phase ${currentPhaseNumber}` }
      - `planning_phase` → return null (orchestrator needs to discover plans first, then call advance to executing_plan)
      - `executing_plan` → return null (orchestrator determines path from plan discovery)
      - `phase_complete` → if more phases, return { command: '/gsd/plan-phase', args: String(currentPhaseNumber + 1) }; if all done, return null
      - `complete` / `failed` → return null

   j. `fail(error: string): void` — set phase to failed, store error message

   k. `setLastCommand(cmd: GsdCommand): void` — record the last command issued (for logging/recovery)

   Note: getNextCommand() returns the COMMAND to issue, not the state to transition to. The orchestrator issues the command, then calls advance() based on the result. This keeps the state machine pure — it doesn't have side effects.
  </action>
