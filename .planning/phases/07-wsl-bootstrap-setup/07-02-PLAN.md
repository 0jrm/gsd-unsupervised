<task type="auto">
  <name>Task 1: Add explicit validation and dry-run modes to setup.sh</name>
  <files>setup.sh</files>
  <action>
  Extend `setup.sh` to support explicit modes via simple flags:
  - `--dry-run` (or `--check`): run all environment and path detection logic, print what would be done (including source/destination paths for rules sync), but do not modify the filesystem beyond temporary checks.
  - `--validate`: run the full bootstrap flow but include post-conditions checks at the end (e.g., confirm `.cursor/rules` exists in the repo, confirm WSL detection succeeded, and confirm that the project root looks healthy).
  - Default (no flags): perform the standard bootstrap behavior from 07-01 (detect, sync rules) and print a short success message plus a hint to run orchestrator smoke tests.
  Ensure that:
  - Invalid or unknown flags print a one-line usage summary and exit non-zero.
  - The script exits non-zero when validation fails and prints a short list of failed checks (e.g., "missing .cursor/rules", "Windows .cursor not found").
  Avoid: complicated flag parsing frameworks; stick to straightforward `case`/`while` parsing that will behave predictably on most Bash installations.
  </action>
  <verify>
  Run:
  - `bash setup.sh --dry-run` and confirm it prints detection details and planned actions without copying or modifying files.
  - `bash setup.sh --validate` and confirm it performs the bootstrap behavior and then reports validation status, exiting non-zero if any required condition is not met.
  </verify>
  <done>
  `setup.sh` supports `--dry-run` and `--validate` modes, with clear output and exit codes that reflect success or failure of the bootstrap and validation steps.
  </done>
</task>

  <action>

    - `npm run lint` or a dedicated `npm run smoke` if configured.

  Avoid: running long or heavy workflows by default in the standard `./setup.sh` mode; the smoke test should be opt-in via `--validate` or a dedicated flag so re-runs stay fast.
  </action>
