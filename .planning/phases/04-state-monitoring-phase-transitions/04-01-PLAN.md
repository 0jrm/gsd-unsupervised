<feature>
  <name>STATE.md Parser</name>
  <files>src/state-parser.ts, src/state-parser.test.ts</files>
  <behavior>
    parseStateMd(content: string) → StateSnapshot | null

    Extracts structured data from the "## Current Position" section of STATE.md:

    Input format (from real STATE.md):
    ```
    ## Current Position

    Phase: 3 of 7 (Cursor Agent Integration)
    Plan: 3 of 3 in current phase
    Status: Phase complete
    Last activity: 2026-03-16 — Completed 03-03-PLAN.md — Phase 3 complete

    Progress: ██████░░░░ 43%
    ```

    Cases:
    - Standard content → { phaseNumber: 3, totalPhases: 7, phaseName: "Cursor Agent Integration", planNumber: 3, totalPlans: 3, status: "Phase complete", lastActivity: "2026-03-16 — Completed 03-03-PLAN.md — Phase 3 complete", progressPercent: 43 }
    - Phase in-progress → { status: "Executing plan", ... }
    - Plan 1 of 1 → { planNumber: 1, totalPlans: 1, ... }
    - Missing "Current Position" section → null
    - Empty/whitespace content → null
    - Malformed phase line (no "of") → null
    - Progress bar without percentage → progressPercent: null
    - No progress line at all → progressPercent: null

    Also export:
    readStateMd(filePath: string) → Promise<StateSnapshot | null>
    - Reads file, returns null if file missing or unparseable
  </behavior>
  <implementation>
    Use regex to extract each field from the "## Current Position" section. Define StateSnapshot as a plain TypeScript interface (no Zod needed — this is read-only data, not user input). Return null if the required section or required fields (phase, plan, status) are missing.

    StateSnapshot interface:
    - phaseNumber: number
    - totalPhases: number
    - phaseName: string
    - planNumber: number
    - totalPlans: number
    - status: string
    - lastActivity: string
    - progressPercent: number | null
  </implementation>
</feature>
