<task type="auto">
  <name>Task 1: Define session log record types and storage contract</name>
  <files>src/session-log.ts</files>
  <action>
Define `SessionLogEntry` in `src/session-log.ts` with: timestamp, goalTitle, phase (command label), phaseNumber, planNumber, sessionId, command, status ('running'|'done'|'crashed'|'timeout'|'verify-failed'|'skipped'), durationMs, error, failureContext. Keep it lightweight—no huge stack traces.

Create or extend `src/session-log.ts` module that:
- Stores log entries in a newline-delimited JSON file at a configurable path (default `./session-log.jsonl` under workspace root per config), keeping the path in one place for later configurability.
- Exposes `appendSessionLog(logPath: string, entry: SessionLogEntry): Promise<void>` that serializes a single entry as JSON plus newline, ensures the parent directory exists, and handles write errors by catching silently (session logging must be best-effort, not crash the daemon).
- Exposes `readSessionLog(logPath: string): Promise<SessionLogEntry[]>` that reads the file and parses entries, returning an empty array if the file is missing or unreadable.

Design for append-only writes: never rewrite or truncate the file, and keep the API surface small so future plans can extend it (e.g. filtering helpers) without breaking the contract.
  </action>

  <action>
Update the orchestrator so that major lifecycle boundaries append structured session log entries using `appendSessionLog(logPath, entry)`.

At minimum, log entries with status 'running' when starting a goal/phase/plan, 'done' on success, 'crashed' or 'timeout' on failure. Include goalTitle, phase (command label), phaseNumber, planNumber, sessionId, command, and optional durationMs/error/failureContext.

Ensure:
- Logging calls are awaited or queued so writes actually land, but they must not block the orchestrator from progressing longer than necessary; if a write fails, catch and log via the existing logger at `warn` level and continue.
- The orchestrator does not change the existing STATE.md contract: session logging augments, not replaces, STATE-based monitoring from Phase 4.
